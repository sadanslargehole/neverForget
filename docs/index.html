Setup

1. Selecting a mode

Messages in private channels will be logged by default if you're using blacklist mode and those channels are not blacklisted!

Whitelist mode makes it so you have to choose channels where unpins are logged. You can mark and un-mark channels for logging with `wlist <add | rm> [channelID].

Blacklist mode makes it so unpins are logged in all channels unless you explicitly mark them as ignored. You can mark and un-mark channels to be ignored with `blist <add | rm> [channelID].

To choose a mode, run `mode set blist for blacklist mode, or `mode set wlist for whitelist mode.

2. Selecting when you want the messages logged

By default messages are logged when they are unpinned.

If you want them logged when they are pinned instead, you can set the bot to log when they are pinned instead

To do this run `mode time <pin | unpin>

3. Setting up a logging channel

To automatically set up a logging channel, run `log setup. This will automatically create a log channel with the following defaults.

  • @everyone will not be allowed to send messages
  • Any role with Manage Messages, Manage Server or Manage Channels will be able to send messages
  • The name #pinned-messages-log will be set by default

You can change the name, permissions, anything you like, as long as the channel is not deleted. If you'd prefer to use your own pre-existing channel, run `log set [channelID].

4. Enable the bot

Run `enable.
